# 解题流程

更新时间：2026-03-29

## 用户入口

默认入口不是脚本列表，而是：

1. 配置 `Claude Code`
2. 在仓库根目录打开会话
3. 运行 `/setup`
4. 提供题目标题、分类、描述、目标和附件

如果字段已经够用，`/setup` 应直接创建 workspace。

## 标准链路

一条题目通常按这个顺序流转：

1. `setup`
2. `challenge-workspace-bootstrap`
3. `ctf-solver-profile`
4. 一个具体题型 skill
5. 需要时调用浏览器自动化或网络搜索
6. `ctf-knowledge-capture`
7. `skill-maintainer`

这是 agent 的内部顺序，不是要求用户手动逐项调用。

当第 7 步准备新增 skill 时，`skill-maintainer` 应先做增长审计：

- 是否和已有 skill 重叠
- 是否只是脚本或测试缺失
- 是否有稳定边界
- 是否已经有 lab / smoke / 回归支撑

审计不过时，不新增 skill，直接退回较小改动。

## 输入

开始一道题，至少应提供这些信息中的大部分：

- 题目标题
- 题目分类
- 题目描述
- URL / 主机 / 端口 / 连接方式
- 附件路径
- 限制条件

## 工作默认值

这个仓库的默认解题姿势是：

- 先做低成本验证
- 先读源码、元信息和结构线索
- 根据证据切方向
- 记录失败路径，不只记录成功路径

## 输出

一次完整流程的产物通常包括：

- `workspaces/<challenge-id>/` 下的题目入口、附件、笔记和工件
- 解题结论
- `knowledge/` 下的 writeup 或 pattern

## 边界

当前流程默认解决的是：

- 单题分析
- 人在回路中的工具调用
- 本地知识回查
- 工件和结论沉淀

当前流程不默认解决的是：

- 批量题目调度
- 自动 flag 提交
- 多 agent 编排平台
